# ROADMAP

## Правило ведения roadmap

- `ROADMAP.md` обязателен к обновлению после каждой завершённой задачи или итерации.
- Выполненные пункты нужно отмечать как завершённые через `[x]`.
- Если задача выполнена частично, нужно явно зафиксировать это в соответствующем пункте или подпункте.
- Нельзя закрывать пункты заранее или отмечать их выполненными без фактической реализации и проверки.

## Общие правила PoC

- Стек: React 18, TypeScript, Vite, styled-components, `@reach/router`, локальные типизированные stubs `@pulse/ui/*`.
- Используются только именованные импорты.
- `any` запрещён.
- Backend не добавляется.
- Новые зависимости добавляются только при явной необходимости.
- Изменения вносятся только в рамках текущей итерации.
- При неоднозначности требований работа останавливается до уточнения.
- После каждой итерации обязательны:
  - `npm run typecheck`
  - `npm run build`
