# Сценарии поиска AddressBook

Документ фиксирует продуктовое поведение верхнего поиска. Выбранное подразделение и текстовый поиск людей — разные режимы экрана кадровой структуры.

## Поиск людей по Enter

| Откуда запущен поиск | Активная вкладка после Enter | `orgFilter` | Результат |
| --- | --- | --- | --- |
| Все контакты | Все контакты | Не передаётся | Все найденные по тексту люди |
| Кадровая структура, корневой экран | Кадровая структура | Не передаётся | Все найденные по компании люди |
| Кадровая структура, открыто подразделение | Кадровая структура | Сбрасывается и не передаётся | Все найденные по компании люди |

При текстовом поиске людей на вкладке кадровой структуры:

- заголовок экрана — «Результаты поиска», как и на вкладке «Все контакты»;
- список людей не ограничивается ранее выбранным подразделением;
- слева отображаются подразделения, в которых найдены люди; до появления отдельного API список строится по подразделениям загруженных сотрудников.

## Выпадающий список поиска

Выпадающий список должен различать людей и организационные структуры.

### Клик по человеку

- переключить активную вкладку на «Все контакты»;
- закрыть результаты общего поиска;
- показать только карточку выбранного человека.

Это отдельное действие и не эквивалентно нажатию Enter: Enter показывает список всех найденных людей на текущей вкладке.

### Клик по организационной структуре

- переключить активную вкладку на «Кадровая структура»;
- открыть выбранное подразделение;
- передать идентификатор подразделения как `orgFilter` при загрузке его сотрудников;
- показать название выбранного подразделения вместо заголовка «Кадровая структура»;
- справа показать кнопку «Наверх» и вложенные подразделения.

Поиск и навигация по организационной структуре будут реализованы после подключения `getGroupHierarchy` и связанных API.

## Правило состояния

`orgFilter` принадлежит режиму просмотра выбранного подразделения. Как только пользователь запускает текстовый поиск людей по Enter, режим выбранного подразделения завершается: `orgFilter` становится `null` и параметр отсутствует в запросе `multiSearch`.

После Enter параметр `q` становится глобальным фильтром и сохраняется при любых переходах по разделам. Текущая вкладка не меняется. Все таблицы применяют сохранённый `q` к своим данным.

Глобальный фильтр завершается только выбором конкретной подсказки:

- выбор человека удаляет `q`, очищает поисковую строку, переключает на «Все контакты» и показывает выбранную карточку;
- выбор оргструктуры удалит `q`, очистит строку и откроет выбранный узел кадровой структуры после подключения API оргструктуры.
